JVM Architecture

Fundamentals Intermediate

Overview

JVM Architecture explains how Java programs are executed internally. It defines how classes are loaded, how memory is managed, and how bytecode is converted into machine-level instructions. Understanding JVM Architecture helps developers improve performance, manage memory efficiently, and debug runtime issues.

Detailed Explanation

The Java Virtual Machine (JVM) is an abstract machine that executes Java bytecode. It allows the same compiled Java program to run on different operating systems without modification.

Main Components of JVM

  • Class Loader Subsystem
  • Runtime Data Areas (Memory)
  • Execution Engine

Key Characteristics

  • Platform independent execution
  • Automatic garbage collection
  • Secure runtime environment
  • Multithreaded execution support
  • High performance using JIT compiler

1. Class Loader Subsystem

Responsible for loading Java class files into memory.

  • Loading: Reads class bytecode
  • Linking: Verifies and prepares memory
  • Initialization: Executes static blocks

2. Runtime Data Areas

Method Area

Stores class metadata, static variables, and method information.

Heap Area

Stores objects and instance variables. Managed by Garbage Collector.

Stack Area

Stores method calls and local variables. Works on LIFO principle.

PC Register

Keeps track of current instruction execution.

3. Execution Engine

  • Interpreter: Executes bytecode line by line
  • JIT Compiler: Converts bytecode into native machine code
  • Garbage Collector: Removes unused objects automatically

Execution Flow

  1. Java source code compiled into bytecode
  2. Class Loader loads class
  3. Bytecode verifier checks security
  4. Execution Engine executes program
  5. Garbage Collector manages memory

Simple Java Example


class Demo {

    public static void main(String[] args) {

        int x = 10;          // Stored in Stack
        Student s = new Student();  // Object stored in Heap

        s.show();
    }
}

class Student {

    void show() {
        System.out.println("Hello JVM");
    }
}

Explanation: Local variables and method calls are stored in Stack memory, while objects created using new keyword are stored in Heap memory.
